## Code Review

Reviewed the full codebase and identified key issues:

- **Critical**: `config.py` (API keys) not in `.gitignore`
- API errors silently returned as model response strings, indistinguishable from real output
- Models called sequentially — could be parallelized
- `API_TIMEOUT` applied to Anthropic only, not OpenAI or Gemini
- Fragile Gemini thinking-model workaround
- No client-side input validation feedback
- History panel loads all saved files on every page load

## Changes Made

### 1. Error Handling (`app.py`)

- `call_claude`, `call_chatgpt`, `call_gemini` now log errors to stderr via `logging` and re-raise instead of returning `"[Model error: ...]"` strings
- Added `logging.basicConfig` at module level
- `_call_models_parallel` catches per-model exceptions and marks those history entries with `"error": True`
- Summary generation failures also log and return a readable fallback string
- Frontend renders error entries with a red left border, darker background, and an "Error" badge — visually distinct from real responses

### 2. Parallelization (`app.py`)

- Added `concurrent.futures.ThreadPoolExecutor` to call all selected models concurrently within each round
- New `_call_models_parallel(models, prompts_by_key, round_num)` helper handles futures, result ordering, and per-model error capture
- Round 0 (initial responses) and all follow-up rounds are parallelized
- Rounds 1+ use a history snapshot taken before the round starts, so all models in a round respond to the same prior context (previously model 2 could see model 1's in-progress round response — a subtle inconsistency)
- Same parallelization applied to `run_followup`

### 3. Copy Buttons (`index.html`, `style.css`)

- Every response card and the summary card have a copy button (Lucide Copy icon)
- On click, icon swaps to the Lucide Check icon for 1.5s as confirmation
- `buildThreadText()` function assembles the full discussion (all rounds, follow-ups, summary) as plain text
- A "Copy All Responses" + "Save Discussion" action bar appears above the discussion area when results load — persistent and visible without scrolling
- Works for both new discussions and loaded history items (all render through the same `renderCard` path)
- SVGs are inlined — no CDN or build step required

### 4. UI Cleanup

- Removed the old bottom-of-page Save/Copy buttons (previously appended dynamically after the last card)
- Replaced with a static action bar (`#discussionActions`) shown/hidden via `display` toggle
- `#copyAllBtn` and `#saveAllBtn` wired up with direct event listeners at page load
